AI Tutor Guide

How the AI tutor works, where your API key goes, and how to verify it yourself.

What the AI tutor does

Lesson pages have annotated sections marked with a sparkle icon. Clicking one opens a sidebar where you can ask an AI math tutor questions about that specific section — definitions, proof strategies, worked examples, and more.

Your API key is never stored, logged, or cached on our server.
It exists in server memory only for the fraction of a second needed to forward your request to the AI provider. After the response is sent back to you, no trace of your key remains on our infrastructure.

What exactly happens

Your Browser

Key stored in browser localStorage. Sent with each chat request.

→ Your key + message, encrypted via TLS

Proof@Home Server

Key in memory for this request only — then gone.

No database. No log file. No disk write. Memory only, request-scoped.

→ Forwarded to provider, encrypted via TLS

AI Provider

Your key in the auth header. Anthropic or OpenAI processes your request.

Why does the key go through the server?

Browser-to-API calls to Anthropic and OpenAI are blocked by CORS policy — the providers do not allow direct requests from web pages. Our server acts as a minimal pass-through proxy: it receives your key, puts it in the authorization header, forwards the request, and returns the response. That's it.

How to get an API key

  • Anthropic (Claude): Sign up at console.anthropic.com, navigate to API Keys, and create one.
  • OpenAI (GPT): Sign up at platform.openai.com, go to API Keys, and create one.

Cost expectations

Typical AI tutor usage costs a few cents per conversation. A focused study session of 20–30 messages usually costs under $0.10 USD with either provider. Both offer usage dashboards so you can track spending.

Verify it yourself

The request handler is open-source. You can read exactly what happens to your key in src/server/handlers/ai_chat.go. We encourage you to audit the code.

The handler reads api_key from the request body, places it in the provider's auth header, and makes a single HTTP call. The key is never assigned to any persistent variable, written to disk, or logged.
