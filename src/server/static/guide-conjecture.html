Conjecture Author Quick Start

Submit new math problems for the community to prove.

1

Install

Install the CLI in one command:

curl -fsSL https://pah.fly.dev/install.sh | sh
2

Sign up & log in

Create an account on the web UI, then authenticate the CLI:

pah auth login
3

Naming Convention

Every conjecture ID must follow the format: {domain}_{topic}_{number}_{prover}

  • domain — lowercase alpha, the broad area (e.g., fin, crn, newman, prob)
  • topic — lowercase alphanumeric, the subtopic (e.g., binom, stoich); optional for single-topic domains
  • number — numeric digits (e.g., 001, 42)
  • prover — lean4 or rocq

Examples: fin_binom_001_lean4, newman_002_rocq, prob_003_lean4

The filename must be {id}.toml and the prover field must match the suffix.

4

Write conjecture TOML

Each conjecture is a TOML file with the required fields:

id = "fin_binom_001_lean4"
title = "Risk-neutral probability lies in (0,1)"
difficulty = "medium"
prover = "lean4"
status = "open"
preamble = "import Mathlib.Tactic"
lemma_statement = "theorem binomial_risk_neutral ..."
skeleton = "theorem binomial_risk_neutral ...\n  sorry"
hints = [
    "The numerator is positive by hypothesis",
]

[dependencies]
lean_toolchain = "leanprover/lean4:v4.16.0"
lake_packages = ["mathlib"]

For Rocq conjectures, use rocq_version and opam_packages under [dependencies].

5

Submit

Submit a directory, tar.gz, or git URL. The CLI auto-seals with submitter NFT metadata:

pah conjecture create ./my-conjectures/
6

View your NFT credit

Your submitter NFT is generated immediately. Browse the gallery to see it.
