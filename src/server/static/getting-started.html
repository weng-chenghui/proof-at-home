Getting Started

Go from zero to your first verified proof contribution in seven steps.

1

Install the CLI

Run the one-line installer to download a pre-built binary:

curl -fsSL https://pah.fly.dev/install.sh | sh

The installer places pah in your PATH. To build from source instead: git clone https://github.com/pah-org/proof-at-home.git && cargo build --release

2

Install a proof assistant

You need at least one proof compiler to verify proofs:

  • Rocq for Rocq/Coq proofs (~10 min with opam)
  • Lean 4 for Lean proofs (~5 min with elan)

Check what's detected: pah tools check

3

Configure

Run the interactive setup wizard:

pah setting set          # Identity, AI provider, budget
pah auth login           # Paste token from web UI

For AI-assisted mode you'll need an API key from Anthropic, OpenAI, or a local Ollama install. Manual proof submission needs no API key.

4

Set your proof budget

Choose how much API credit to allocate per run:

pah setting set budget

Read and accept the legal agreement, then pick an amount ($1–$10 or custom). This caps the API cost for a single contribution run.

Typical costs: ~$0.05/proof with Haiku, ~$0.50 with Sonnet, ~$2.00 with Opus. A $2 budget can prove dozens of easy conjectures.

5

Prove your first conjectures

AI-assisted — LLM generates proofs, compiler verifies:

pah contribution run

Manual — you write proofs, compiler verifies:

pah proof submit prob_001 ./my_proof.v

The CLI fetches open conjectures, generates or accepts a formal proof, compiles it with rocq c or lean to verify correctness, submits the results, and archives everything with a SHA-256 hash.

6

Check your contributions

See your contributions and their status:

pah contribution list
7

View in the NFT Gallery

Every contribution generates NFT-compatible metadata. Browse all contributions and their verification hashes in the gallery.
