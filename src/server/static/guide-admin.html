Site Admin Setup Guide

Deploy your own Proof@Home instance from scratch. Follow these steps to create a data repository, configure the server, and start accepting contributions.

1

Create a data repository

The data repo is the single source of truth for conjectures, strategies, contributions, certificates, and expositions. Create a new git repository with this directory structure:

your-data-repo/
├── conjectures/     ← conjecture JSON files
├── contributions/   ← proof packages (created by contributors)
├── certificates/    ← review certificates (created by certifiers)
├── expositions/     ← proof explanations (created by expositors)
└── strategies/      ← proving strategy markdown files

See examples/data-repo/ in the Proof@Home source tree for a working template.

2

Host it on a git forge

Push your data repo to GitHub, GitLab, or keep it as a local bare repository. The server will clone and pull from this remote.

# GitHub
make setup-github

# GitLab
make setup-gitlab
3

Configure environment variables

Set these environment variables before starting the server:

Variable Required Description
GIT_DATA_REPO_URL Yes Remote URL or local path of the data repo
GIT_DATA_REPO_PATH No Local clone path (default: ./data)
GIT_FORGE_TYPE Yes github, gitlab, or local
GIT_FORGE_TOKEN For GitHub/GitLab API token for creating PRs
GIT_FORGE_PROJECT For GitHub/GitLab owner/repo (GitHub) or project ID (GitLab)
WEBHOOK_SECRET Recommended HMAC secret for webhook verification
4

Seed conjectures

Add conjecture JSON files to the conjectures/ directory. Each file defines one challenge:

{
  "id": "my-conjecture-001",
  "title": "Sum of two primes",
  "description": "Prove that every even integer greater than 2 ...",
  "difficulty": "medium",
  "prover": "lean4",
  "status": "open"
}

The server loads all files from conjectures/ on startup.

5

Add strategies

Strategy files live in strategies/ and use TOML frontmatter delimited by +++:

+++
name = "induction-on-nat"
kind = "tactic"
prover = "lean4"
description = "Structural induction on natural numbers"
+++

## Strategy

Use `induction n` to split into base case and
inductive step. Simplify with `simp` and `omega`.
6

Deploy & configure webhook

Deploy the server (for example with Fly.io):

fly deploy

Then configure your git forge to send a webhook on push to main:

  • URL: https://your-instance.fly.dev/webhooks/git
  • Content type: application/json
  • Secret: your WEBHOOK_SECRET value

The server will auto-sync whenever you push to the data repo.

7

Verify

Visit the web UI and confirm that data appears on each page:
