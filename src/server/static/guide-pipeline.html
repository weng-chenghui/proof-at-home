Content Pipeline

How educational content flows from conjectures to courses. Each stage supports three creation methods: AI-assisted, pair-proved, and manual.

Pipeline Overview

1. Conjecture
→
2. Proof
→
3. Exposition
→
4. Lesson
→
5. Series

Three Methods per Resource

Method Who does AI work Cost stdin needed
api-assisted (default) pah calls AI provider directly Tracked No
pair-proved User pastes into Claude/ChatGPT Free Yes
manual User writes everything by hand Free Yes

Pair-Proved Workflow

For users who have access to Claude but no API key configured:

1

Export the prompt

pah lesson export --conjectures id1,id2,id3 > prompt.txt
2

Paste into Claude

Copy the contents of prompt.txt into Claude or ChatGPT. The AI generates a complete lesson.md file.

3

Pipe the result back

cat lesson.md | pah lesson create --stdin --method pair-proved

Lesson Commands

pah lesson list                              # List all lessons
pah lesson get <id>                          # Get lesson details
pah lesson create --topic <topic> \
  --conjectures id1,id2,id3                  # AI-assisted creation
pah lesson create --stdin --method manual     # From stdin
pah lesson export --conjectures id1,id2       # Export prompt

Series Commands

pah series list                              # List all series
pah series get <id>                          # Get series details
pah series create --lessons id1,id2,id3      # AI-assisted creation
pah series export --lessons id1,id2           # Export prompt

Lesson Format

Lessons use Markdown with YAML frontmatter at lessons/{id}/lesson.md:

---
lesson_id: my-lesson
title: "My Lesson Title"
topic: algebra
difficulty: medium
conjecture_ids: [conj_001, conj_002]
published: true
ai_annotations:
  - zone: "## Section Heading"
    context: "What this section covers"
    suggestions:
      - "A suggested question"
---
# My Lesson Title

Markdown body with $\LaTeX$ math...

Series Format

Series use the same pattern at series/{id}/series.md:

---
series_id: intro-verification
title: "Introduction to Verification"
lesson_ids:
  - getting-started
  - newman-lemma
published: true
---
# Series Title

Markdown body with learning path...

Full Pipeline Example

# 1. List conjectures
pah conjecture list

# 2. AI-assisted proof run
pah contribution run --by prove-coq-lemma

# 3. Generate exposition
pah exposition create --for newman_001

# 4. Compose lesson from conjectures
pah lesson create --topic rewriting-systems \
  --conjectures newman_001,newman_002,newman_003

# 5. Package into series
pah series create --lessons how-to-create-a-lesson,newman-lemma

Troubleshooting

No API key configured

Run pah setting set for the interactive wizard, or export PAH_API_KEY. Alternatively, use the pair-proved workflow above.

Lesson not appearing

Ensure published: true in frontmatter, and that the lesson_id matches the directory name. The server cache rebuilds from git on push.

Proof verification failure

Check prover version with pah tools check. Ensure dependencies match the conjecture's requirements.
