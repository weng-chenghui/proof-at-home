Submit Conjectures

A guide for conjecture authors: write formal statements and submit them for the community to prove.

1

Write conjecture JSON files

Each conjecture is a JSON file with the following fields:

{
  "id": "prob_004",
  "title": "Your conjecture title",
  "difficulty": "easy|medium|hard",
  "prover": "rocq|lean4",
  "status": "open",
  "preamble": "Require Import Arith.",
  "lemma_statement": "Lemma foo : forall n, n + 0 = n.",
  "hints": ["Try induction on n"],
  "skeleton": "Lemma foo : forall n, n + 0 = n.\nProof.\n  (* your proof here *)\nAdmitted."
}
  • id — unique identifier (e.g. prob_004)
  • prover — rocq or lean4
  • preamble — imports and setup code for the proof assistant
  • lemma_statement — the formal statement to prove
  • hints — optional hints for provers
  • skeleton — a proof template with holes to fill in
2

Install the CLI

Build from source:

git clone https://github.com/pah-org/proof-at-home.git
cd proof-at-home
cargo build --release

Then configure it with ./target/release/proof-at-home init and set the server URL.

3

Authenticate

Sign up at this server's signup page, copy your auth token from settings, then run:

./target/release/proof-at-home login

Paste the token when prompted.

4

Submit your conjectures

Three submission methods are supported:

# Submit a directory of conjecture JSON files
pah conjecture create ./my-conjectures/

# Submit a tar.gz archive
pah conjecture create /tmp/conjectures.tar.gz

# Submit a git repo URL (server clones it)
pah conjecture create https://github.com/example/conjectures.git

The server validates each conjecture, commits them to a git branch, creates a pull request, and generates submitter NFT metadata.

5

Track your conjectures

Browse your submitted conjectures and their status:

# List all conjectures
pah conjecture list

# View details of a specific conjecture
pah conjecture get prob_004

Or browse them on the Conjectures page.

6

Your submitter NFT

Each submission generates NFT metadata crediting you as the conjecture author. The metadata records:

  • Your username and batch ID
  • Number of conjectures submitted
  • Difficulty levels and proof assistants
  • Git commit SHA for provenance
